Git Tutorial

Git_Branch_command

1. Download Git

Install Git from git-scm.com.

2. Set Username and Email

git config --global user.name "username"
git config --global user.email "username@gmail.com"

# Check configured name and email
git config --global user.name
git config --global user.email

3. Initialize Git

git init     # Initialize an empty Git repo

ls -lart     # Show hidden files (like .git folder)

4. Git File Structure

  • Untracked
  • Unmodified
  • Modified
  • Staged

File flow:

Untracked → Staged → Committed → Unmodified

Unmodified → Modified → Staged → Committed → Unmodified

Commit a file manually:

git commit
# Inside the commit screen:
press i (insert)
type: Initial commit
press ESC
type: :wq (save and quit)

5. Common Git Commands

git status           # Show untracked/modified files
touch about.html     # Create a blank file
git add -A           # Add all changes to staging
git commit -m "Added HTML file"
git checkout contact.html    # Discard changes to a file
git checkout -f              # Force reset all files

git log              # Show commit history
git log -p -2        # Show last 2 commits with changes

git diff             # Compare working dir with staging area
git diff --staged    # Compare staging area with last commit

git commit -a -m "Quick commit, skip staging"

git rm fileName      # Remove file from working dir
git rm --cached file # Untrack file (keep in storage)

git status -s        # Short status view

6. .gitignore Examples

/contact.html      # Ignore only in current dir
*contact.html      # Ignore all matching files
gitignor/          # Ignore the folder named gitignor

Create the file:

touch .gitignore

7. Git Branching

git branch new_feature      # Create a new branch
git branch                  # List all branches
git checkout new_feature    # Switch to new_feature branch
git checkout main           # Switch back to main

git merge new_feature       # Merge changes to main

git checkout -b api_free    # Create and switch to new branch

# Create a file in the new branch:
touch api.php

# Note: api.php won't appear in main branch until merged.

8. Push Repository to GitHub

      # Initialize Git (if not done)
      git init
      
      # Add and commit files
      git add .
      git commit -m "Initial commit"
      
      # Add GitHub repo as remote
      git remote add origin https://github.com/your-username/your-repo-name.git
      
      # Push to GitHub
      git push -u origin main

Note: Replace your-username and your-repo-name with your actual GitHub info.

Use a personal access token if prompted for a password.

Git Branch Management Commands

Create New Branch

To create a new branch:

git branch feature/new-feature

Example: git branch feature/new-feature

Switch Branch

To switch to another branch:

git checkout feature/new-feature

Example: git checkout feature/new-feature

Merge Branch

To merge a branch into the current branch:

git merge feature/new-feature

Example: git merge feature/new-feature

Delete Branch

To delete a branch locally:

git branch -d feature/new-feature

Example: git branch -d feature/new-feature

Git Add

To add changes to the staging area:

git add index.html

Example: git add index.html

Git Commit

To commit changes to the local repository:

git commit -m "Add new feature to homepage"

Example: git commit -m "Add new feature to homepage"
